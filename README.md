# Claude Usage — Stream Deck plugin

Displays your current Claude usage limits on a Stream Deck key as a color-coded gauge:
percent used, and time until the window resets.

![preview](com.mrjefftang.claude-usage.sdPlugin/imgs/plugin/marketplace.png)

## What it does

1. **Authenticates with OAuth** — the same PKCE flow Claude Code uses. You sign in with
   your Claude account in the key's settings (Property Inspector); tokens are stored in the
   plugin's global settings and refreshed automatically.
2. **Polls `/api/oauth/usage`** every 3 minutes (the endpoint is aggressively rate-limited,
   so 3 min is the safe floor) to fetch the current 5-hour and 7-day usage and limits.
3. **Renders a gauge** — a 270° arc colored green → amber → orange → red as you approach the
   limit, the percent used in the center, and the time to reset at the bottom.

Each key can show a different window: **5-hour session**, **7-day (all models)**,
**7-day Sonnet**, or **7-day Opus** — selectable in the Property Inspector.

Tap a key to force an immediate refresh.

## Requirements

- Stream Deck app **6.5+** (the plugin uses the Node.js plugin runtime declared in the
  manifest; Stream Deck supplies Node 20).
- A Claude account (Pro/Max/Team) — the usage endpoint reflects your subscription limits.

## Install

```bash
./install.sh
```

This copies `com.mrjefftang.claude-usage.sdPlugin` into the Stream Deck plugins folder and
restarts the Stream Deck app. Then:

1. Drag the **Claude Usage → Usage Limit** action onto a key.
2. Open the key's settings, pick a window, and click **Sign in with Claude**.
3. Approve in the browser, copy the code Claude shows you, paste it back, and click
   **Complete sign-in**.

To install manually, copy the `.sdPlugin` folder to:

- **macOS:** `~/Library/Application Support/com.elgato.StreamDeck/Plugins/`
- **Windows:** `%APPDATA%\Elgato\StreamDeck\Plugins\`

…then quit and relaunch Stream Deck.

## Project layout

```
com.mrjefftang.claude-usage.sdPlugin/
  manifest.json        plugin + action declaration (Node 20 runtime)
  plugin.js            Stream Deck WebSocket client, polling, event loop
  lib/oauth.js         Claude OAuth (PKCE): authorize URL, token exchange, refresh
  lib/usage.js         GET /api/oauth/usage with the required headers
  lib/render.js        SVG gauge → data URI
  ui/inspector.html    Property Inspector (sign-in + window picker)
  imgs/                PNG icons (generated by tools/gen-icons.mjs)
  node_modules/ws      WebSocket client (only runtime dependency)
tools/gen-icons.mjs    regenerates the PNG icons
```

## Notes on auth & privacy

- The OAuth client ID, authorize/token URLs, and scopes are the public Claude Code values.
  No secrets are embedded (PKCE means no client secret is needed).
- Your access/refresh tokens are stored via Stream Deck's `setGlobalSettings` (local to your
  machine, in the Stream Deck app's settings). **Sign out** in the Property Inspector clears
  them.
- The usage request sends a `claude-code/<version>` User-Agent. Per Anthropic, the bare
  bucket without that header is rate-limited hard; with it, a 3-minute poll is safe.
